## cAdvisor 개별 컨테이너 메트릭 미수집

### 증상
- `container_memory_working_set_bytes` 메트릭에서 `id="/"`(루트 cgroup)만 수집됨
- `name`, `container` 등 개별 컨테이너 구분 라벨 없음
- 호스트 전체 합산값만 보이고, 개별 컨테이너(Spring, Redis 등) 메트릭 구분 불가

### 환경
- Ubuntu 22.04+ (cgroup v2)
- Docker v29+
- Grafana Alloy v1.13.2 (내장 cAdvisor v0.47.0 포크)
- cAdvisor는 독립 컨테이너가 아닌 Alloy `prometheus.exporter.cadvisor` 내장 방식

### 원인
- Docker v29에서 스토리지 레이아웃이 `overlay2` → `overlayfs`로 변경됨
- Alloy v1.13.2에 내장된 cAdvisor(v0.47.0 포크)가 이 변경을 지원하지 못함
- Alloy 로그에서 다음 에러 확인:
  ```
  Failed to create existing container: /system.slice/docker-<hash>.scope:
  failed to identify the read-write layer ID for container "<hash>".
  open /rootfs/var/lib/docker/image/overlayfs/layerdb/mounts/<hash>/mount-id:
  no such file or directory
  ```
- GitHub Issue: https://github.com/grafana/alloy/issues/5021

### 시도했으나 효과 없었던 것들
1. `store_container_labels = true` 설정 → 근본 원인이 아님
2. `docker_only = true` 설정 → 근본 원인이 아님
3. 호스트 볼륨 마운트 추가 (`/rootfs`, `/sys`, `/var/lib/docker` 등) + `privileged: true` → 근본 원인이 아님

### 해결
- Alloy 이미지를 `v1.14.0-rc.1`로 업그레이드
  - PR #5447에서 내장 cAdvisor가 v0.54.1로 업데이트되어 Docker v29 지원
  - 2026-03-09 시점 v1.14.0은 RC 단계이며 정식 릴리즈 미출시
- `docker_only = true` 설정으로 Docker 컨테이너 메트릭만 수집
- 업그레이드 후 에러 소멸, `name` 라벨 포함한 개별 컨테이너 메트릭 정상 수집 확인

### 부수 발견
- `docker compose up -d`는 바인드 마운트된 설정 파일의 **내용 변경을 감지하지 않음**
  - 이미지, 포트, 볼륨 선언, 환경변수 등 compose 정의만 비교
  - 설정 파일만 바뀐 경우 `docker compose restart`가 별도로 필요
  - deploy.sh에 `docker compose restart` 추가하여 해결

### 후속 조치
- v1.14.0 정식 릴리즈 시 RC에서 stable로 전환 필요
- 다른 노드(app, redis, caddy)의 Alloy도 동일 이슈 있을 수 있으므로 순차 적용 검토
